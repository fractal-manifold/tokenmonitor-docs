From box to wall
in about three minutes.

Install the broker, power the device, join its captive portal, pair from your AI agent. Live usage flows within ~60 seconds.

01

Install the broker on your laptop.

The broker is a tiny daemon that reads your CLIs' usage logs and exposes them over the LAN. Ships in three languages — pick whichever toolchain you already have.

# Go ≥ 1.21 — adds cwm-mcp to $GOPATH/bin go install github.com/fractal-manifold/cwm-mcp/cmd/cwm-mcp@latest
# Python 3.10+ — installs into an isolated venv via pipx pipx install cwm-mcp-py
# Node 18+ — global install via npm npm install -g cwm-mcp-js
# macOS / Linux — auto-detects toolchain, sets up launchd / systemd curl -fsSL https://github.com/fractal-manifold/cwm-mcp/raw/main/cwm-mcp-launcher/install.sh | sh
Verify it's running The broker listens on localhost:9787. Hit curl localhost:9787/health — you should see {"role":"broker","status":"ok"}.

02

Power the device.

USB-C, 5 V / 1 A. Any wall charger or USB hub will do — the device draws < 4 W peak with the backlight at full.

First boot lands on the Connecting → Needs Config screen and announces the local IP address plus a 6-digit pairing code.

Power consumption ≈ 3 W with backlight at 80%, ≈ 0.4 W with backlight off in standby. Battery (where fitted) holds ~3 hours of operation for power cuts.
03

Connect to its captive portal.

The device exposes an open Wi-Fi SSID — your phone or laptop auto-opens the portal at 192.168.4.1.

  • SSID: ClaudeWallMonitor-XXXX (last 4 = MAC suffix)
  • Enter your home Wi-Fi SSID + password
  • Pick your city (controls weather + sunrise/sunset)
  • Submit → device reboots into "Waiting for setup"
Why is it an open network? Only the Wi-Fi join is unauthenticated, and only for the ~30 s of provisioning. The actual broker channel that follows is HMAC + AES-CTR encrypted with your passphrase-derived PSK.

04

Pair from your AI agent. The magic moment.

Install the plugin (one line — see the plugin page), then ask Claude Code, Codex or Gemini to configure your device:

> /cwallmonitor:configure

The agent does the rest:

  • Discovers your device on the LAN via mDNS
  • Asks for the 6-digit code shown on screen
  • Generates a passphrase, derives the PSK locally
  • Pushes broker URL + key, device reboots
What about Codex / Gemini? Same skill, same UX, different CLI. See plugin install for the per-CLI snippet.
05

You're done.

The device lands on the dashboard. Within ~60 seconds the broker's first poll completes and live usage starts flowing — session, weekly and (optionally) design-time cards refresh every 2.4 s.

Mount it. Forget it. Glance at it.

Standard 86 × 86 mm wall switch box. Two screws, one USB-C cable. Standby kicks in after 20 min idle; tap anywhere to wake.
