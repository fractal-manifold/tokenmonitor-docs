Dashboard

What each number means.

The default screen is three stacked cards plus a single ambient strip. Designed to be read from 2–3 m away in under a second.

  1. 1

    Current — 5-h session window

    How much of your active 5-hour Claude / Codex / Antigravity quota you've burned. Right-side projection is the AI's own forecast for end of window.

  2. 2

    Weekly — monthly cap pace

    Percent of the monthly limit consumed, with the day-of-month projection to the right. Yellow when you're trending over.

  3. 3

    Design — week's average

    Optional smaller card. Tracks long-running threads (Claude's "Design" mode). Hidden when zero so it never adds noise.

  4. 4

    Ambient strip

    Sunrise · sunset · temperature · battery. Lives under the header, stays lit in standby. Polls Open-Meteo hourly.

  5. 5

    Last-sync indicator

    The status word at the bottom (Whispering · Generating · Synthesizing) animates only while the broker has fresh data. Stale > 60 s → fades to dim.

Percentages or dollars? Depends on how you pay. On a Pro / Max subscription the cards show the percentage of your plan quota used — the number that actually matters when there's no per-token bill. On a pay-as-you-go API key there's no quota to fill, so the device shows an estimated $ instead — your local token counts priced at public list rates (an estimate, not the invoice).

Standby

Quiet after 20 minutes.

Backlight ramps to 0% after 20 min of no touch. The last values persist in framebuffer along with the ambient strip — at 0% backlight they're invisible but the panel doesn't blank.

Tap anywhere — or press the PWR side button — to wake. The dashboard fades back in over 600 ms. A short PWR press also sends it back to standby on demand.

Why so long? 20 min is empirically about one focused work block. Shorter and the device blanks mid-task; longer and you stop noticing it. Both bounds are tweakable in Settings.

Buttons

Two side buttons.

Everything day-to-day is touch, but the two physical side buttons cover the moments the screen can't help you — sleeping, restarting, and starting over.

PWR — sleep, wake & restart

  • Short press — toggle standby: blank the screen, or wake it.
  • Hold ~2 s — restart the device. Nothing is erased — it just reboots, the way you'd power-cycle anything that's acting up. Wi-Fi, city, theme and pairing all come back exactly as they were.

BOOT — factory reset

  • Hold ~10 s — start a factory reset. A confirmation screen appears first; nothing is erased until you tap Erase (or Cancel to back out).
  • Confirming wipes your settings and pairing — Wi-Fi, city, brightness, providers, alert volume and the broker passphrase — then reboots into first-boot setup, ready to provision again.
  • The device's own identity and security state (its ID and update anti-rollback floor) are preserved, so it re-pairs cleanly without losing its place in the update chain.
Restart first, reset last If something looks wrong, hold PWR to restart before reaching for BOOT — a reboot fixes most transient glitches and keeps your setup intact. Factory reset is the clean-slate option. (Holding BOOT while powering on is a separate developer mode — see the FAQ.)

Settings

Tap the gear, bottom-right.

The gear icon on the dashboard's footer band opens Settings. Fields are grouped into five sections so the list scans at a glance instead of becoming one long flat enumeration.

Providers

  • Claude Code · Codex · Antigravity (toggle each on / off)

Display

  • Auto-rotate (Enabled / Disabled)
  • Rotate interval (10..300 s)
  • Theme (Day / Night / Auto)
  • Day brightness · Night brightness

Network

  • City (controls weather + Auto-theme sunrise / sunset)
  • Wi-Fi SSID + password
  • Broker URL (override the auto-discovered one)
  • Passphrase (rotates PSK — broker side updates via skill)

Audio

  • Alert volume (0..100%) with four test buttons for live preview

About

  • Device ID · Firmware version · IP address · Broker URL (read-only diagnostics)

Edits land on the editor screen with a slider + pencil for exact input. Cancel restores the previous value. The About rows are not editable — they show live device state for troubleshooting.

Alerts

Four events. Hysteresis-armed.

The device fires four kinds of alert. Each is armed with a hysteresis band so it can't oscillate at the boundary — you get a single beep + flash, not a chatter loop.

Battery low

< 20%

Battery icon turns warning-orange. Soft chime on the hour.

icon · orangechime · soft

Battery critical

< 10%

Wakes the device from standby. Repeating beep until USB power is reconnected.

full wakebeep · repeat

Limit overshoot

→ > 100%

Predicted to exceed the session or weekly cap before reset. Card outline pulses gold; one chime.

card · pulsechime · once

5-h window rollover

@ rollover

Marks the moment Claude's 5-hour session window resets. Subtle bell.

flash · purplebell · soft

Themes

Day · Night · Auto.

Three modes. Auto follows sunrise/sunset for your configured city — with a ±90 s hysteresis so a passing cloud doesn't trigger a swap. Switch any time from Claude Code, Codex or Antigravity:

Custom panel

Draw your own data.

Swipe up from the dashboard to reach an extra screen that shows whatever you want — a line or bar chart, a pie/donut, a key-value table or plain text. A small script on your laptop writes a JSON file; the broker serves it to the device over the same authenticated LAN link; the device draws it. Edit the file and the screen refreshes within ~20 s — nothing to recompile or reflash.

It's an opt-in feature, off by default. Turn it on from the device Settings, or from your CLI:

> /tokenmonitor:settings panel_enabled=true

The data is entirely yours — the device just renders the file your script writes. Point the broker at it with a [panel] section in tokenmonitor.toml.

# tokenmonitor.toml [panel] file = "~/.local/state/tokenmonitor/panel.json" # panel.json — rewritten by your script { "version": 1, "title": "Session", "tiles": [{ "type": "line", "title": "Tokens/min", "series": [{ "name": "in", "color": "#c15f3c", "points": [12,18,9,24,31] }] }] }

Control plane

Two skills do the heavy lifting.

/tokenmonitor:configure pairs and provisions. /tokenmonitor:theme switches themes. Both ship with every CLI plugin.
